**##    Description**

|   `call_me_maybe` is a short project involving function-calling with an LLM: Qwen.
|   The project is straight-forward: Give a prompt file and a function definition file to the LLM, and it will find functions to use.

**##    Instructions**

|   You can use the following command to install the .venv and requirements:

```sh

make build

```

|   If your PC has a GPU card, the program will be a lot faster.

|   You can use the following command to run the program:

```sh

make run ARGS="--functions_definition <func_file.json> --input <prompt.json>"
uv run --no-sync python -m src --functions_definition <func_file.json> --input <prompt.json>

```

|   Parameters:
    ->  functions\_definition: Path to the function definition json file.
    ->  input: Path to the input json file.
    ->  output: Functions generated by the LLM will be saved to 'data/output' + filename.

|   You can use the following command to debug the program:

```sh

make debug ARGS="--functions_definition <func_file.json> --input <prompt.json>"

```

|   Note: func\_file.json and prompt.json are required. If they are invalid, the program exits with code 1.
|   Note: If a function to a prompt is not found, the program continues, but said prompt will an inexistent function.

|   You can use the following command to clean the workspace:

```sh

make clean

rm -rf ~/.cache/huggingface # in case you ran `uv run python ...` instead of `make run`.

```

|   The program supports several models and has other argument options you can use:

```txt

--functions_definition=<func_file.json> -> path to the function definition file
--input=<prompt.json>                   -> path to the prompts file
--model=(Qwen/SmolLM2/opt-125)          -> allows different models (default: Qwen3-06B)
--output=(output.json)                  -> writes to a specific file, then outputs it to data/output (default: data/output + output.json)
--max-token=(any int)                   -> how many tokens allowed for use (default: 2048)

```

|   Here are some available models:

```txt

|   Qwen: Qwen3-06B, Qwen3-06B-Base, Qwen2.5-0.5B
|   SmolLM2: SmolLM2-135M, SmolLM2-135M-Instruct
|   Meta: opt-125M
|   CompactAI: CompactAI-O/TMLM-Haiku-2.3

#   Note: Meta, SmolLM2 models are almost unusable. The output.json they return is completely false. Stick to Qwen's model if you want accuracy and not complete BS.
#   Note: Qwen3-06B-Base is actually faster than Qwen3-06B, and provides the same output.json.

```
